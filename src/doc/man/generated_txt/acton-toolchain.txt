ACTON-TOOLCHAIN(1)

NAME
       acton-toolchain — Manage project-selected Acton toolchains

SYNOPSIS
       acton toolchain [options] command

DESCRIPTION
       Manage Acton releases installed side-by-side for projects that define
       [toolchain] in Acton.toml.

       Project toolchains do not replace the global acton binary in PATH. They
       are stored under:

           ~/.acton/toolchains/<acton-version>/acton

       Use acton up when you want to update the global Acton binary. Use acton
       toolchain install when you want to install the Acton release required by
       a project.

PROJECT SELECTION
       Project toolchains are selected from the optional [toolchain] section in
       Acton.toml:

           [toolchain]
           acton = "0.3.0"
           tolk = "1.3.0"

       Both fields are exact versions.

       If acton is set, Acton selects that release. If tolk is also set, the
       selected Acton release must ship that exact Tolk version.

       If only tolk is set, Acton uses the currently running binary when it
       already ships that Tolk version. Otherwise, Acton selects the newest
       known Acton release that ships the requested Tolk version.

SUBCOMMANDS
   acton toolchain install
       Install the Acton toolchain selected for this project.

      Synopsis
          acton toolchain install [acton-version]

      Options
          acton-version
              Exact Acton version to install.

              If omitted, Acton resolves the toolchain from the current
              project’s Acton.toml.

          Install an exact Acton release:

              acton toolchain install 0.3.0

          From a project root, install the toolchain resolved from Acton.toml:

              acton toolchain install

          This is the recommended CI preinstall step:

              acton toolchain install
              acton test

          If the current binary already satisfies the selected project
          toolchain, the command succeeds as a no-op. If the selected release
          is already installed, it also succeeds as a no-op.

   acton toolchain list
       List installed and known Acton project toolchains.

      Synopsis
          acton toolchain list

          List installed side-by-side toolchains and known Acton/Tolk mappings:

              acton toolchain list

          Yanked releases remain visible so diagnostics can explain why a
          pinned version cannot be selected.

   acton toolchain which
       Print the executable selected for the current project.

      Synopsis
          acton toolchain which

          Print the executable path selected for the current project:

              acton toolchain which

          If the selected toolchain is missing, the command fails with the
          exact acton toolchain install command to run.

   acton toolchain resolve
       Resolve the Acton toolchain selected for the current project.

      Synopsis
          acton toolchain resolve

          Print machine-readable resolution details:

              acton toolchain resolve

          The JSON output includes the source of the selection, Acton version,
          bundled Tolk version, whether the selected binary is current or
          installed, and whether installation is required.

   acton toolchain remove
       Remove an installed side-by-side Acton toolchain.

      Synopsis
          acton toolchain remove acton-version

      Options
          acton-version
              Exact Acton version to remove.

          Remove an installed side-by-side Acton release:

              acton toolchain remove 0.3.0

          Removal asks for confirmation in an interactive terminal.

ONE-COMMAND SELECTOR
       Use a leading +<version> selector to run one project command with a
       specific Acton toolchain:

           acton +0.3.0 test
           acton +0.3.0 build

       The selector is an Acton version, not a Tolk version. It must be the
       first argument after acton, and it is not supported for commands such as
       acton up, acton help, acton toolchain, or shell completion generation.

TOOLCHAIN INDEX
       Acton resolves known Acton/Tolk mappings from the toolchain index.

       The index is fetched from the Acton GitHub repositories and cached under
       ~/.acton/toolchains/index.json. If the cache is fresh, Acton uses it
       without a network request. If a stale cache cannot be refreshed, Acton
       continues with the cached index and prints a warning.

       If no index cache is available and the index cannot be fetched, commands
       that need release metadata fail. An explicitly pinned Acton version that
       is already installed can still run from its local release.json metadata
       unless that metadata marks it as yanked.

       Yanked releases are rejected during resolution when yank metadata is
       available.

DISPLAY OPTIONS
       --color when
           Control when to use colored output.

           Possible values: auto, always, never

PROJECT OPTIONS
       --manifest-path path
           Path to the Acton.toml file to use for this invocation.

           Use this when running the command outside the project directory or
           when the manifest lives at a non-default location.

       --project-root path
           Path to the project root to use for configuration discovery and
           relative defaults.

           This conflicts with --manifest-path.

EXIT STATUS
       o  0: The selected toolchain operation completed successfully.

       o  1: Resolution failed, the selected release is unavailable or yanked,
          installation failed, or removal failed.

EXAMPLES
       1. Install the toolchain selected by the current project:

              acton toolchain install

       2. Install a specific side-by-side Acton release:

              acton toolchain install 0.3.0

       3. Print known and installed toolchains:

              acton toolchain list

       4. Run tests with a one-command Acton override:

              acton +0.3.0 test
