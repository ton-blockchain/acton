Acton is currently released as a public beta. This page explains which
platforms are first-class, which surfaces are beta, and what "stable" means in
Acton's release language.

<Callout type="warning">
    "Stable" on this page refers to the release channel and support
    expectations, not to a claim that the entire product has already exited
    public beta.
</Callout>

## Release channels

- **Stable channel**: the latest numbered GitHub release. This is the default
  install/update path and the channel to use for day-to-day development.
- **Trunk channel**: prerelease builds installed via `acton up --trunk`. Use
  this to try new features early, but expect rough edges and occasional
  breaking changes.

## Platform tiers

| Platform / environment                        | Tier               | Notes                                                                                                                                                                |
|-----------------------------------------------|--------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| macOS ARM64 (`aarch64-apple-darwin`)          | First-class        | Official release artifact, documented install path, and release automation coverage.                                                                                 |
| macOS x86_64 (`x86_64-apple-darwin`)          | First-class        | Official release artifact, documented install path, and release automation coverage.                                                                                 |
| Linux x86_64 GNU (`x86_64-unknown-linux-gnu`) | First-class        | Official release artifact, documented install path, regular PR CI coverage, and release automation coverage.                                                         |
| Linux ARM64 GNU (`aarch64-unknown-linux-gnu`) | First-class        | Official release artifact, documented install path, and release automation coverage.                                                                                 |
| Native Windows                                | Unsupported        | Native Windows shells and workflows are not supported. Use WSL instead.                                                                                              |
| Windows via WSL                               | Beta / best-effort | Recommended way to use Acton on Windows today. Use Ubuntu 20.04 or newer inside WSL and follow the Linux install path; there is still no dedicated Windows artifact. |
| Other source-built targets                    | Beta / best-effort | Not part of the published binary matrix; validation is lighter and regressions can surface later.                                                                    |

## What first-class means

- Versioned release artifacts are published for the platform.
- The platform is part of the documented installation story.
- Regressions on that platform are treated as supported bugs.
- The platform is exercised by at least one automated validation path before
  release.

## What beta means

- The surface may work, but compatibility is not guaranteed across releases.
- Validation is lighter than on the first-class matrix.
- You may need prerelease builds, source builds, or platform-specific
  workarounds.
- Fixes are best-effort until the surface moves into the first-class matrix.
