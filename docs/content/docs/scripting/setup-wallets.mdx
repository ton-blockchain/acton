Wallets are essential for interacting with the real blockchain in Acton. They enable you to deploy contracts, send transactions, and verify smart contract source code.

## Wallet Storage

Acton supports two levels of wallet configuration:

1.  **Local Wallets (`wallets.toml`)**: Project-specific wallets. Stored in the project root.
2.  **Global Wallets (`~/.acton/wallets/global.wallets.toml`)**: Shared across all your Acton projects.

When you create a project using `acton new` or `acton init`, Acton automatically creates a symbolic link `global.wallets.toml` pointing to your global wallets for easy access.

<Callout type="warn">
    Both `wallets.toml` and `global.wallets.toml` contain sensitive information (mnemonics). Acton automatically adds them to your `.gitignore`, but you should always be careful not to commit them.
</Callout>

## Wallet Configuration

Wallets are defined in a TOML file under the `[wallets]` section. Each wallet gets a unique name that you'll use to reference it in scripts and verification commands.

Here's a basic wallet configuration:

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
workchain = 0
keys = { mnemonic = "word1 word2 word3 ... word24" }
```

This configuration creates a wallet named `deployer` that:
- Uses the V5R1 wallet version
- Operates in workchain 0 (the basechain)
- Stores the mnemonic directly in the config file

## Providing Mnemonic Phrases

Acton supports three methods for providing wallet mnemonics.

### 1. Direct Mnemonic (Default)

The simplest way is to store the mnemonic directly in your `wallets.toml` or global config. This is the default when creating a new wallet via `acton wallet new`.

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
keys = { mnemonic = "word1 word2 word3 ... word24" }
```

### 2. Environment Variables

For higher security, especially in CI/CD environments, you can store mnemonics in environment variables:

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
keys = { mnemonic-env = "WALLET_MNEMONIC" }
```

Set the environment variable before running your script:

```bash
export WALLET_MNEMONIC="word1 word2 word3 ... word24"
acton script deploy.tolk --broadcast
```

### 3. File References

Alternatively, store the mnemonic in a separate file and reference it:

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
keys = { mnemonic-file = ".secrets/deployer.txt" }
```

## Creating and Managing Wallets

### Creating a New Wallet

Use the `acton wallet new` command to generate a new testnet wallet:

```bash
# Create a local wallet
acton wallet new --name deployer --local

# Create a global wallet
acton wallet new --name shared --global
```

### Listing Wallets

To see all available wallets (local and global) and their types:

```bash
acton wallet list
```

You can also check their balances on the testnet:

```bash
acton wallet list --balance
```

## Address Safety Checks

To prevent accidental use of wrong wallets, configure expected addresses:

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
workchain = 0
keys = { mnemonic = "..." }

[wallets.deployer.expected]
address-testnet = "0QD36XRy1ISfSB8zmlSQKBsa5bmeixAWwD6vUUhOXXur8ZLd"
address-mainnet = "UQD36XRy1ISfSB8zmlSQKBsa5bmeixAWwD6vUUhOXXur8XSS"
```

Before executing any operations, Acton verifies that the wallet derived from your mnemonic matches the expected address. This safety check helps catch configuration mistakes before spending real TON.

## Multiple Wallets

Local wallets override global wallets if they share the same name. This allows you to have a "default" global wallet but override it for a specific project.

```toml title="wallets.toml"
[wallets.deployer]
kind = "v5r1"
keys = { mnemonic-env = "DEPLOYER_MNEMONIC" }

[wallets.deployer.expected]
address-testnet = "0QD36XRy1ISfSB8zmlSQKBsa5bmeixAWwD6vUUhOXXur8ZLd"

[wallets.admin]
kind = "v4r2"
keys = { mnemonic-env = "ADMIN_MNEMONIC" }
```

## Using Wallets in Scripts

Once configured, reference wallets by name in your scripts using `net.wallet()`:

```tolk title="deploy.tolk"
import "lib/emulation/network"
import "lib/io"

fun main() {
    val deployer = net.wallet("deployer");
    println1("Deployer address: {}", deployer.address);

    // Use deployer.address for contract deployment
    // ...
}
```

Run the script with the `--broadcast` flag to use the real wallet:

```bash
acton script deploy.tolk --broadcast
```

Without `--broadcast`, `net.wallet()` creates a local test wallet with the same name, perfect for testing scripts locally before running them on the real blockchain.

## Security Best Practices

1. **Never commit mnemonics**: Always use environment variables or files listed in `.gitignore`
2. **Use expected addresses**: Configure `expected.address-testnet` and `expected.address-mainnet` to catch configuration errors
3. **Test first**: Run scripts without `--broadcast` to test locally before interacting with the real blockchain
4. **Separate wallets**: Use different wallets for testnet and mainnet operations
5. **Start with testnet**: Always test your scripts on testnet before running them on mainnet
