title: "Getting Started"
description: "Start Acton local development node, use faucet, and run a minimal local workflow"
---

The local development node is available through the `acton localnet` command group.

## Start the Node

```bash
acton localnet start
```

By default, the node listens on port `5411` (or `[localnet].port` from `Acton.toml`).

You can verify that the server is running:

```bash
curl -s "http://127.0.0.1:5411/api/v2/getMasterchainInfo"
```

## Fund an Address

Use the built-in faucet command:

```bash
acton localnet airdrop UQA_ftKIJsHEAE_UgtFOUK15hPzycZooFuUr8duyY9T3kwwM
```

Set a custom amount:

```bash
acton localnet airdrop UQA_ftKIJsHEAE_UgtFOUK15hPzycZooFuUr8duyY9T3kwwM --amount 50
```

## Use a Custom Port

```bash
acton localnet start --port 3010
```

Then target the same port for CLI helpers:

```bash
acton localnet airdrop UQA_ftKIJsHEAE_UgtFOUK15hPzycZooFuUr8duyY9T3kwwM --port 3010
```

## Simulate API Latency

Use `--response-delay-ms` when you need local API requests to behave more like
a slower remote provider:

```bash
acton localnet start --response-delay-ms 300
```

This delays `/api/v2`, `/api/v3`, and `/api/emulate/v1` responses. Streaming
subscriptions, control endpoints, and the explorer UI are not delayed.

You can change the delay while the node is running:

```bash
curl -s -X POST "http://127.0.0.1:5411/acton_setNetworkConditions" \
  -H "content-type: application/json" \
  -d '{"response_delay_ms":300}'
```

Use `{"response_delay_ms":0}` to disable the delay. The current value is
included in `/acton_nodeInfo`.

You can also store the same default in `Acton.toml`:

```acton-toml title="Acton.toml"
[localnet]
response-delay-ms = 300
```

## Stop the Node

Press `Ctrl+C` in the terminal where the node is running.

<Callout type="info">
If you use `--dump-state`, the snapshot is saved during graceful shutdown.
</Callout>
