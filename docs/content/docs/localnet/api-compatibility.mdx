`/api/v2/getShardAccountCell` accepts `address` and optional `seqno`, and returns
a TonCenter `tvm.cell` object with base64-encoded BOC bytes. The same method is
also available through the v2 JSON-RPC entrypoints: `POST /api/v2` and
`POST /api/v2/jsonRPC`.

`/api/v2/sendBoc`, `/api/v2/sendBocReturnHash`, and the v2 JSON-RPC `sendBoc`
methods accept external-in messages only. Use `/acton_sendInternalMessage` for
local development flows that need to inject a raw internal message.

## TonCenter v3 Coverage Matrix

Acton local development node currently supports 11 TonCenter v3 endpoints:

| Method | Endpoint                        | Available in local node | Purpose                                                              |
|--------|---------------------------------|-------------------------|----------------------------------------------------------------------|
| `GET`  | `/api/v3/traces`                | ✅                       | Returns transaction trace tree for a transaction hash.               |
| `GET`  | `/api/v3/accountStates`         | ✅                       | Returns account states with `interfaces` and token metadata.         |
| `GET`  | `/api/v3/addressInformation`    | ✅                       | Returns account state in `V2AddressInformation` shape.               |
| `GET`  | `/api/v3/transactions`          | ✅                       | Returns transactions with TonCenter v3 blockchain filters.           |
| `GET`  | `/api/v3/transactionsByMessage` | ✅                       | Returns transactions filtered by message hash/body hash/opcode.      |
| `GET`  | `/api/v3/pendingTransactions`   | ✅                       | Returns pending transactions filtered by account and trace id.       |
| `POST` | `/api/v3/message`               | ✅                       | Sends an external message in BOC and returns message hash fields.    |
| `POST` | `/api/v3/runGetMethod`          | ✅                       | Runs a get method of a smart contract with an explicit stack.        |
| `GET`  | `/api/v3/jetton/masters`        | ✅                       | Returns jetton master metadata with optional filters and pagination. |
| `GET`  | `/api/v3/jetton/wallets`        | ✅                       | Returns jetton wallet metadata with optional filters and pagination. |
| `GET`  | `/api/v3/nft/items`             | ✅                       | Returns NFT items with optional filters, pagination, and metadata.   |

`/api/v3/transactions` currently supports only single-value query filters for `account` and `exclude_account`; `/api/v3/pendingTransactions` supports only single-value query filters for `account` and `trace_id`. `/api/v3/transactionsByMessage` filters such as `msg_hash`, `body_hash`, `opcode`, and `direction` are also single-value in local development node (no repeated multi-value form yet).

`/api/v3/message` accepts external-in messages only. Use
`/acton_sendInternalMessage` to inject a raw internal message in local tooling.

No other v3 endpoints are exposed by the local development node at the moment.

## Control API

| Method | Endpoint                      | Request                                                                                  | What it does                                                                              |
|--------|-------------------------------|------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------|
| `POST` | `/acton_fundAccount`          | JSON body: `{"address":"<ADDR>","amount":<NANOTONS>}`                                    | Credits address with local funds and mines the transfer transaction.                      |
| `GET`  | `/acton_getAddressName`       | Query: `?address=<ADDR>`                                                                 | Returns current alias for an address (or `null` if not set).                              |
| `POST` | `/acton_setAddressName`       | JSON body: `{"address":"<ADDR>","name":"<ALIAS>"}`                                       | Sets or updates local alias for an address.                                               |
| `GET`  | `/acton_getCompilerAbi`       | Query: `?code_hash=<HEX_OR_BASE64_HASH>`                                                 | Returns the registered compiler ABI for a code hash, or `null` when no ABI is registered. |
| `POST` | `/acton_registerCompilerAbis` | JSON body: `{"entries":[{"code_hash":"<HEX_OR_BASE64_HASH>","compiler_abi":{...}},...]}` | Registers one or more compiler ABIs for later explorer/API lookups.                       |
| `GET`  | `/acton_nodeInfo`             | No params                                                                                | Returns uptime, latest block seqno, and the active local or remote state source.          |
| `POST` | `/acton_dumpState`            | JSON body: `{"path":"<SNAPSHOT_PATH>"}`                                                  | Writes a JSON state snapshot without stopping the server.                                 |
| `POST` | `/acton_loadState`            | JSON body: `{"path":"<SNAPSHOT_PATH>"}`                                                  | Replaces the current node state with a JSON state snapshot.                               |
| `GET`  | `/acton_getStateSource`       | No params                                                                                | Returns current state source used for account resolution (`Local` or `Remote`).           |
| `POST` | `/acton_setStateSource`       | JSON enum payload (`"Local"` or `{"Remote": {...}}`)                                     | Switches account state source at runtime.                                                 |
| `POST` | `/acton_setShardAccount`      | JSON body: `{"address":"<ADDR>","shard_account":"<BASE64_BOC>"}`                         | Replaces the selected account state with a base64-encoded `ShardAccount` BOC.             |
| `POST` | `/acton_sendInternalMessage`  | JSON body: `{"boc":"<BASE64_BOC>"}`                                                      | Sends a base64-encoded internal message BOC through the local internal queue.             |

<Callout type="warn">
  Control routes are intended for local development tooling and should not be exposed publicly. The
  local node does not implement authentication for these routes. Loopback-only CORS helps browser
  usage, but it is not an access-control mechanism.
</Callout>

<Callout type="info">
  Rate limiting applies only to `/api` routes. Control routes remain available even when
  `--rate-limit` or `[localnet].rate-limit` is enabled.
</Callout>
