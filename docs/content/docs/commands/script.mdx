---
title: "Running Tolk scripts"
description: "Learn how to use the script command to execute standalone Tolk files for experimentation, deployment, and blockchain interaction"
---

The `script` command executes standalone Tolk files as scripts. Scripts can be used for learning Tolk, experimenting with smart contracts, deploying to the blockchain, and interacting with live contracts. Unlike tests, scripts define a `main()` function as the entry point and can interact with the real TON blockchain when using the `--broadcast` flag.

## Basic Usage

```bash
acton script my_script.tolk
```

This command executes `my_script.tolk` in the local emulator environment.

## Command Options

### Input Options

| Option          | Type      | Description                                |
|-----------------|-----------|--------------------------------------------|
| `<PATH>`        | File path | Path to the Tolk script file to execute    |

### Network Options

| Option                 | Type   | Description                                                                                                   |
|------------------------|--------|---------------------------------------------------------------------------------------------------------------|
| `--broadcast`          | Flag   | Execute transactions on real blockchain                                                                       |
| `--net <NET>`          | String | Target network for broadcast: `testnet` (default), `mainnet`, `localnet`, or `custom:<name>`                  |
| `--fork-net <NETWORK>` | String | Fork blockchain state from network for local execution (`mainnet`, `testnet`, `localnet`, or `custom:<name>`) |
| `--api-key <API_KEY>`  | String | TonCenter API key for blockchain queries                                                                      |

### Debug Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--debug`               | Boolean   | Enable debug mode with DAP server                         |
| `--debug-port <PORT>`   | Integer   | Debug server port (default: 8080)                         |

### Cache Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--clear-cache`         | Flag      | Clear compilation cache before running                    |

### Output Options

| Option          | Type | Description                                                                |
|-----------------|------|----------------------------------------------------------------------------|
| `--show-bodies` | Flag | Show decoded message bodies in printed transaction trees when ABI is known |

## Script Structure

Every Tolk script must define a `main()` function with no arguments:

```tolk title="hello.tolk"
import "@acton/io"

fun main() {
    println("Hello, TON!");
}
```

Scripts have access to all functions available in tests, including:
- Contract deployment and interaction
- Message sending
- Get-method execution
- Blockchain state queries
- etc.

## Examples

### Local Experimentation

Run a script in the local emulator (no real TON spent):

```bash
# Basic script execution
acton script experiment.tolk

# Show decoded transaction bodies in println(txs) / println(result)
acton script experiment.tolk --show-bodies

# Clear cache before running
acton script experiment.tolk --clear-cache
```

### Deploying to Blockchain

Deploy a contract to the selected network using wallets:

```bash
# Deploy to testnet (default)
acton script deploy.tolk --broadcast

# Deploy to mainnet
acton script deploy.tolk --broadcast --net mainnet

# Deploy to localnet
acton script deploy.tolk --broadcast --net localnet
```

<Callout type="error" title="Broadcast sends real transactions">
With `--broadcast`, transactions are sent to the selected network. On `mainnet`/`testnet` this spends real TON. On `localnet`, local node balances are used.
</Callout>

Example deployment script:

```tolk title="deploy.tolk"
import "@acton/io"
import "@acton/emulation/network"
import "@acton/build/build"

import "@contracts/types"
import "@wrappers/Counter"

fun main() {
    // 1. Load the configured wallet (searches wallets.toml and global wallets)
    val deployer = net.wallet("deployer");
    val counter = Counter.fromStorage(Storage {
        id: 8,
        counter: 0,
    });

    val result = counter.deploy(deployer.address, ton("0.05"));
    result.wait(); // optional wait for deployment when broadcasting

    println1("Deployed counter to {}", counter.address);
}
```

### Querying Live Contracts

Query state from real blockchain contracts:

```bash
# Query testnet contract
acton script query.tolk --fork-net testnet

# Query mainnet contract with API key
acton script query.tolk --fork-net mainnet --api-key YOUR_API_KEY
```

Example query script:

```tolk title="query.tolk"
import "@acton/io"
import "@acton/emulation/network"

fun main() {
    val contractAddress = address("EQDt7LL...");
    val currentValue: int = net.runGetMethod(contractAddress, "currentCounter");
    println1("Counter value: {}", currentValue);
}
```

## Script Execution Model

Scripts are executed as smart contracts with these characteristics:

1. **Entry Point**: The `main()` function is treated as a get-method with ID=0
2. **Exit Code**: Script terminates with exit code equal to the `main` execution exit code (capped at 255)
3. **State**: State is not preserved between script runs

## Execution Modes

| Mode                             | Blockchain State   | Wallets      | Real TON Spent |
|----------------------------------|--------------------|--------------|----------------|
| Local (default)                  | Local emulator     | Test wallets | No             |
| `--fork-net testnet`             | Real testnet state | Test wallets | No             |
| `--broadcast`                    | Local emulator     | Real wallets | Yes            |
| `--broadcast --fork-net testnet` | Real testnet state | Real wallets | Yes            |

Custom networks follow the same model through `custom:<name>`.
