---
title: "Compiling Tolk smart contracts"
description: "Learn how to use the compile command to build Tolk source code into TVM bitcode"
---

The `compile` command compiles Tolk source files into TVM (TON Virtual Machine) bitcode using official Tolk compiler. This process transforms high-level Tolk code into executable bitcode that can run on the TON blockchain.

## Basic Usage

```bash
acton compile contract.tolk
```

This command compiles `contract.tolk` and displays the resulting bitcode information.

## Command Options

### Input Options

| Option          | Type      | Description                                |
|-----------------|-----------|--------------------------------------------|
| `<PATH>`        | File path | Path to the Tolk source file to compile    |

### Output Options

| Option                      | Type      | Description                                          |
|-----------------------------|-----------|------------------------------------------------------|
| `--json`                    | Flag      | Output compilation result as JSON                    |
| `--base64-only`             | Flag      | Output only the base64-encoded bitcode               |
| `--boc <BOC>`               | File path | Save compiled bitcode to binary BoC file             |
| `--fift <FIFT>`             | File path | Save Fift code representation to file                |
| `--source-map <SOURCE_MAP>` | File path | Generate source map file (enables debug compilation) |

### Cache Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--clear-cache`         | Flag      | Clear compilation cache before compiling                  |

### Saving compilation artifacts

```bash
# Save bitcode to BOC file
acton compile contract.tolk --boc contract.boc

# Save Fift code representation
acton compile contract.tolk --fift contract.fif

# Generate source map for debugging
acton compile contract.tolk --source-map contract.json --boc contract.boc
```

## Cache Behavior

Acton uses compilation caching to speed up rebuilds:

- **Cache location**: `.acton/cache/` in your project directory
- **Cache clearing**: Use `--clear-cache` to force recompilation
- **Cache invalidation**: Automatically invalidated when source files change
