## Synopsis

`acton toolchain` [_options_] _command_

## Description

Manage Acton releases installed side-by-side for projects that define
`[toolchain]` in `Acton.toml`.

Project toolchains do not replace the global `acton` binary in `PATH`. They are
stored under:

```text
~/.acton/toolchains/<acton-version>/acton
```

Use `acton up` when you want to update the global Acton binary. Use
`acton toolchain install` when you want to install the Acton release required by
a project.

## Project Selection

Project toolchains are selected from the optional `[toolchain]` section in
`Acton.toml`:

```toml
[toolchain]
acton = "0.3.0"
tolk = "1.3.0"
```

Both fields are exact versions.

If `acton` is set, Acton selects that release. If `tolk` is also set, the
selected Acton release must ship that exact Tolk version.

If only `tolk` is set, Acton uses the currently running binary when it already
ships that Tolk version. Otherwise, Acton selects the newest known Acton
release that ships the requested Tolk version.

Install an exact Acton release:


From a project root, install the toolchain resolved from `Acton.toml`:

```bash
acton toolchain install
```

This is the recommended CI preinstall step:

```bash
acton toolchain install
acton test
```

If the current binary already satisfies the selected project toolchain, the
command succeeds as a no-op. If the selected release is already installed, it
also succeeds as a no-op.

### acton toolchain list

List installed and known Acton project toolchains.

#### Synopsis

`acton toolchain list`

List installed side-by-side toolchains and known Acton/Tolk mappings:

```bash
acton toolchain list
```

Yanked releases remain visible so diagnostics can explain why a pinned version
cannot be selected.

### acton toolchain which

Print the executable selected for the current project.

#### Synopsis

`acton toolchain which`

Print the executable path selected for the current project:

```bash
acton toolchain which
```

If the selected toolchain is missing, the command fails with the exact
`acton toolchain install` command to run.

### acton toolchain resolve

Resolve the Acton toolchain selected for the current project.

#### Synopsis

`acton toolchain resolve`

Print machine-readable resolution details:

```bash
acton toolchain resolve
```

The JSON output includes the source of the selection, Acton version, bundled
Tolk version, whether the selected binary is current or installed, and whether
installation is required.

## One-command Selector

Use a leading `+<version>` selector to run one project command with a specific
Acton toolchain:

```bash
acton +0.3.0 test
acton +0.3.0 build
```

The selector is an Acton version, not a Tolk version. It must be the first
argument after `acton`, and it is not supported for commands such as
`acton up`, `acton help`, `acton toolchain`, or shell completion generation.

## Toolchain Index

Acton resolves known Acton/Tolk mappings from the toolchain index.

The index is fetched from the Acton GitHub repositories and cached under
`~/.acton/toolchains/index.json`. If the cache is fresh, Acton uses it without a
network request. If the cache cannot be refreshed, Acton can continue with the
best available cached or bundled index and prints a warning.

Yanked releases are rejected during resolution when yank metadata is available.

## Exit Status

- `0`: The selected toolchain operation completed successfully.
- `1`: Resolution failed, the selected release is unavailable or yanked,
  installation failed, or removal failed.

## Examples

1. Install the toolchain selected by the current project:

   ```bash
   acton toolchain install
   ```

2. Install a specific side-by-side Acton release:

   ```bash
   acton toolchain install 0.3.0
   ```

3. Print known and installed toolchains:

   ```bash
   acton toolchain list
   ```

4. Run tests with a one-command Acton override:

   ```bash
   acton +0.3.0 test
   ```
