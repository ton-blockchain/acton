---
title: "Running tests"
description: "Learn how to run Tolk smart contract tests using the test command"
---

The `test` command runs your Tolk smart contract tests. It discovers and executes test functions from files ending with `.test.tolk`, providing detailed output and various reporting options.

## Basic Usage

```bash
acton test
```

This command discovers and runs all test files in project root and its subdirectories.

## Command Options

### Discovery Options

| Option                 | Type      | Description                                           |
|------------------------|-----------|-------------------------------------------------------|
| `<PATH>`               | File path | Test file or directory to run (default: project root) |
| `--filter <PATTERN>`   | String    | Regex pattern to filter test names                    |
| `--exclude <PATTERNS>` | Array     | Glob patterns to exclude test files                   |
| `--include <PATTERNS>` | Array     | Glob patterns to include only test files              |

### Execution Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--fail-fast`           | Flag      | Stop executing tests after the first failure              |

### Reporter Options

| Option                 | Type  | Description                                                                |
|------------------------|-------|----------------------------------------------------------------------------|
| `--reporter <FORMATS>` | Array | Test output format(s)                                                      |
| `--show-bodies`        | Flag  | Show decoded message bodies in printed transaction trees when ABI is known |

**Supported reporters:**
- `console` — Detailed console output with code context (default)
- `teamcity` — TeamCity CI integration (minimal output, best combined with console)
- `junit` — JUnit XML reports for external tools

### Debug Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--debug`               | Boolean   | Enable debug mode                                         |
| `--debug-port <PORT>`   | Integer   | Debug server port                                         |
| `--backtrace <LEVEL>`   | String    | Enable stack traces ("full")                              |

### Coverage Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--coverage`            | Boolean   | Enable code coverage                                      |
| `--format <FORMAT>`     | String    | Coverage output format ("lcov")                           |

### Profiling Options

| Option                       | Type   | Description                                            |
|------------------------------|--------|--------------------------------------------------------|
| `--snapshot <FILE>`          | String | Create JSON snapshot of gas usage statistics           |
| `--baseline-snapshot <FILE>` | String | Compare gas usage with baseline snapshot file          |
| `--fail-on-diff`             | Flag   | Exit with non-zero code when compared snapshot differs |

<Callout type="info">
If both `--snapshot` and `--baseline-snapshot` are provided, Acton runs in
comparison mode and does not overwrite the snapshot file.
</Callout>

<Callout type="info">
`--fail-on-diff` requires `--baseline-snapshot`.
</Callout>

### Mutation Testing Options

| Option                    | Type    | Description                                       |
|---------------------------|---------|---------------------------------------------------|
| `--mutate`                | Flag    | Run tests in mutation testing mode                |
| `--mutate-contract <ID>`  | String  | Contract ID from `Acton.toml` to mutate           |
| `--disable-rule <RULE>`   | Array   | Disable specific mutation rules                   |

### JUnit Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--junit-path <PATH>`   | String    | JUnit XML output directory                                |
| `--junit-merge`         | Boolean   | Merge all suites into one file                            |

### Cache Options

| Option                  | Type      | Description                                               |
|-------------------------|-----------|-----------------------------------------------------------|
| `--clear-cache`         | Flag      | Clear test compilation cache                              |

### Tracing and UI Options

| Option                    | Type    | Description                                                           |
|---------------------------|---------|-----------------------------------------------------------------------|
| `--save-test-trace [DIR]` | String  | Save transaction traces to directory (`.acton/traces` if DIR omitted) |
| `--ui`                    | Flag    | Open test results in browser                                          |
| `--ui-port <PORT>`        | Integer | UI server port (default: `12344`)                                     |

## Examples

### Basic test execution

```bash
# Run all tests in project root
acton test

# Run tests in specific directory
acton test ./contracts

# Run specific test file
acton test my_contract.test.tolk
```

### Filtering tests

```bash
# Run tests matching pattern
acton test --filter "test-wallet"

# Run only integration tests
acton test --filter "test-integration.*"
```

### Different output formats

```bash
# Console output with TeamCity integration
acton test --reporter console,teamcity

# Show decoded message bodies in printed transaction trees
acton test --show-bodies

# Generate JUnit XML reports
acton test --reporter junit --junit-path ./reports

# Generate coverage report
acton test --coverage --format lcov
```

### Mutation testing

```bash
# Run mutation tests for the 'wallet' contract
acton test --mutate --mutate-contract wallet

# Disable specific rules
acton test --mutate --mutate-contract wallet --disable-rule remove_assert
```

### Gas profiling and snapshots

```bash
# Create baseline gas snapshot
acton test --snapshot .acton/gas-baseline.json

# Compare current run with baseline snapshot
acton test --baseline-snapshot .acton/gas-baseline.json

# CI mode: fail build on benchmark drift
acton test --baseline-snapshot .acton/gas-baseline.json --fail-on-diff
```

### Debug mode

```bash
# Start debugging session
acton test --debug --debug-port 8080

# Enable full backtraces
acton test --backtrace full
```

## Test Discovery

Acton automatically discovers test files by:
- Finding files ending with `.test.tolk`
- Searching in the specified directory and subdirectories

<Callout type="info">
Path behavior:

- If `<PATH>` is omitted, Acton discovers tests from project root.
- Global project root/manifest resolution is described in
  [`Project Root Resolution`](/docs/acton-toml/#project-root-resolution).
- If `<PATH>` is provided and relative, it is resolved from your current working directory.
- Use `--project-root` to run from another directory while keeping project-relative defaults in the target project.
</Callout>

## Configuration

Test behavior can be configured via `Acton.toml`:

```toml
[test]
# Default reporter
reporter = ["console"]

# Enable coverage by default
coverage = true

# Filter pattern
filter = "test-.*"

# JUnit output settings
junit-path = "reports"
junit-merge = false
```
