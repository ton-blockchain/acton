The `hooks` command helps you manage project-local Git hooks through Git's
`core.hooksPath` setting. Acton expects hooks to live in `.githooks/` in the
resolved project root.

## Commands

### `acton hooks new`

Creates a `.githooks/` scaffold in the resolved project root.

```bash
acton hooks new
```

If `--template` is omitted, Acton opens an interactive selector with two
options:

- `empty` creates an empty `.githooks/pre-commit`
- `default` creates `.githooks/pre-commit` with a starter hook:

```sh
#!/bin/sh
set -e
acton check
acton fmt --check
```

#### Options

| Option                        | Type   | Description                      |
|-------------------------------|--------|----------------------------------|
| `--template <empty\|default>` | String | Hook scaffold template to create |

#### Examples

```bash
# Choose template interactively
acton hooks new

# Create an empty pre-commit hook
acton hooks new --template empty

# Create a default-made pre-commit hook
acton hooks new --template default
```

<Callout type="warn">
If `.githooks/` already exists, `acton hooks new` fails instead of overwriting
existing hooks.
</Callout>

---

### `acton hooks install`

Configures the Git repository for the resolved project root to use `.githooks/`:

```bash
acton hooks install
```

Internally this runs:

```bash
git config core.hooksPath .githooks
```

On success, Acton prints:

```text
Git hooks installed successfully from .githooks
```

---

### `acton hooks status`

Reports whether the Git repository for the resolved project root already points
`core.hooksPath` to `.githooks`:

```bash
acton hooks status
```

Possible outputs:

```text
Git hooks are installed
```

or:

```text
Git hooks are not installed
```

---

### `acton hooks uninstall`

Removes the `core.hooksPath` override from the Git repository for the resolved
project root:

```bash
acton hooks uninstall
```

Internally this runs:

```bash
git config --unset core.hooksPath
```

On success, Acton prints:

```text
Git hooks uninstalled successfully
```

## Recommended Workflow

```bash
acton hooks new --template default
acton hooks install
acton hooks status
```

After that, Git runs hooks from `.githooks/` for the current repository.
