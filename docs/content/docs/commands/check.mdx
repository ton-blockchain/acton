---
title: "Checking Tolk contracts"
description: "Lint Tolk files and contracts"
---

The `acton check` command runs the Tolk linter on your project. By default, it checks all contracts from `Acton.toml` and any `*.test.tolk` files in the workspace.

## Usage

```bash
acton check
acton check <contract-id>
acton check <path-to-file.tolk>
```

## Options

| Option             | Type   | Description                                              |
|:-------------------|:-------|:---------------------------------------------------------|
| `--fix`            | Flag   | Automatically apply available fixes (console mode only). |
| `--json`           | Flag   | Output results as JSON.                                  |
| `--sarif <PATH>`   | String | Write SARIF report to file.                              |
| `--explain <CODE>` | String | Explain a rule.                                          |

For setup and configuration, see [Linting](/docs/linting).
For the full rules catalog, see [Linter Rules](/docs/linting/rules).

## Behavior Notes

- `--fix` applies edits only in non-JSON mode. With `--json`, diagnostics still include fix suggestions, but files are not changed.
- `acton check <path-to-file.tolk>` runs the file in single-file mode with test-style lint settings. In this mode, `E014 (acton-import-in-contract)` is relaxed.
- `acton check <contract-id>` keeps normal contract lint settings from `Acton.toml` (including `E014`).

## Examples

```bash
# Check the whole project
acton check

# Check a specific contract by id from Acton.toml
acton check counter

# Check a single file
acton check counter.tolk

# Write SARIF report
acton check --sarif .acton/reports/lint.sarif

# Print JSON to stdout and write SARIF file in the same run
acton check --json --sarif .acton/reports/lint.sarif
```
