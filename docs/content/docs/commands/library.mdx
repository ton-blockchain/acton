The `library` command allows you to manage on-chain libraries in the blockchain. It provides subcommands for publishing new libraries and fetching existing ones.

<Callout title="New to libraries?">
    Check out the [Libraries](/docs/advanced/libraries) guide for a
    comprehensive overview and step-by-step usage scenarios.
</Callout>

## Publish Library

The `publish` subcommand allows you to deploy a contract or arbitrary code as a library to the Masterchain.

### Basic Usage

```bash
acton library publish [CONTRACT_ID]
```

This command initiates the publication process for the specified contract. If no options are provided, it will interactively prompt for necessary information.

### Command Options

| Option                  | Type     | Description                                                                                  |
|-------------------------|----------|----------------------------------------------------------------------------------------------|
| `[CONTRACT_ID]`         | Argument | The ID of the contract in `Acton.toml` to publish                                            |
| `--code <CODE>`         | String   | Base64 or Hex code to publish instead of compiling a contract                                |
| `--duration <DURATION>` | String   | Duration to store the library (e.g., `100d`, `1y`, `31536000s`)                              |
| `--wallet <WALLET>`     | String   | Name of the wallet to use for the transaction                                                |
| `--net <NET>`           | String   | Network to use (`testnet`, `mainnet`, `localnet`, or `custom:<name>`), defaults to `testnet` |
| `--api-key <KEY>`       | String   | TonCenter API key for blockchain interactions                                                |
| `--amount <AMOUNT>`     | String   | Explicit TON amount to send (overrides suggested calculation)                                |
| `-y, --yes`             | Flag     | Skip confirmation prompts                                                                    |
| `--local`               | Flag     | Save library info to local `libraries.toml`                                                  |
| `--global`              | Flag     | Save library info to global `global.libraries.toml`                                          |

### Examples

#### Interactive publish

```bash
acton library publish math
```

#### Non-interactive publish with specific duration

```bash
acton library publish math --duration 365d --wallet my-wallet --net mainnet
```

#### Publishing arbitrary code

```bash
acton library publish --code "te6cckEBAQEAAgAAAEysuc0=" --duration 100d
```

## Fetch Library

The `fetch` subcommand retrieves a library from the blockchain by its hash.

### Basic Usage

```bash
acton library fetch <HASH>
```

### Command Options

| Option                | Type      | Description                                                                                  |
|-----------------------|-----------|----------------------------------------------------------------------------------------------|
| `<HASH>`              | Argument  | The SHA-256 hash of the library code                                                         |
| `--disasm`            | Flag      | Disassemble the fetched code and print TASM                                                  |
| `-o, --output <FILE>` | File path | Save fetched code to file (binary if `.boc` extension, else Base64)                          |
| `--json`              | Flag      | Output result as JSON                                                                        |
| `--net <NET>`         | String    | Network to use (`testnet`, `mainnet`, `localnet`, or `custom:<name>`), defaults to `testnet` |
| `--api-key <KEY>`     | String    | TonCenter API key for blockchain interactions                                                |

### Examples

#### Fetch and print code

```bash
acton library fetch 8F452D7A4DFD74066B682365177259ED05734435BE76B5FD4BD5D8AF2B7C3D68
```

#### Fetch and disassemble

```bash
acton library fetch <HASH> --disasm
```

#### Save code to file

```bash
# Save as binary BoC
acton library fetch <HASH> -o library.boc

# Save as Base64 text
acton library fetch <HASH> -o library.txt
```

#### JSON Output

```bash
acton library fetch <HASH> --json
```

Success output:
```json
{
  "success": true,
  "code_boc64": "te6ccgEBAQEAKwAAUQAAkcYpqaMXHEovatIzyqsSjoI7NLF4PF9pY6VdLnAhd6XRCFTWD2lA..."
}
```

Error output:
```json
{
  "success": false,
  "error": "Library not found"
}
```

## Library Info

The `info` subcommand displays detailed information about a deployed library, including its configuration, on-chain balance, and estimated remaining storage time.

### Basic Usage

```bash
acton library info [LIBRARY_NAME]
```

If `LIBRARY_NAME` is not provided, it will show a selection list of available libraries.

### Command Options

| Option            | Type     | Description                                   |
|-------------------|----------|-----------------------------------------------|
| `[LIBRARY_NAME]`  | Argument | Name of the library to show info for          |
| `--api-key <KEY>` | String   | TonCenter API key for balance check           |

### Examples

```bash
acton library info math
```

The output includes:

- **Deployed at**: Initial publication timestamp.
- **Last top-up**: Timestamp of the latest top-up operation. This value is used as the reference point for storage runway calculation.
- **Remaining**: Estimated storage runway computed from current balance and elapsed time since `last_topup_timestamp`.

If `Remaining` is exhausted, Acton prints a warning that top-up is needed urgently. This does not always mean the account is already frozen, but it indicates elevated freeze risk.

## Library Top-up

The `topup` subcommand allows you to extend the storage life of a library by sending TON to its account.

### Basic Usage

```bash
acton library topup [LIBRARY_NAME]
```

### Command Options

| Option                  | Type     | Description                                                     |
|-------------------------|----------|-----------------------------------------------------------------|
| `[LIBRARY_NAME]`        | Argument | Name of the library to top up                                   |
| `--duration <DURATION>` | String   | Duration to extend storage for (e.g., `100d`, `1y`)             |
| `--amount <AMOUNT>`     | String   | Explicit TON amount to send (overrides duration calculation)    |
| `--wallet <WALLET>`     | String   | Name of the wallet to use for the transaction                   |
| `--api-key <KEY>`       | String   | TonCenter API key for blockchain interactions                   |
| `-y, --yes`             | Flag     | Skip confirmation prompts                                       |

### Examples

#### Top up for 1 year (interactive)

```bash
acton library topup math --duration 1y
```

#### Send specific amount

```bash
acton library topup math --amount 5.5 --wallet my-wallet
```

After a successful top-up transaction, Acton updates `last_topup_timestamp` in the corresponding `libraries.toml`/`global.libraries.toml` entry.

## Libraries Metadata File

When library metadata is saved to `libraries.toml` (or `global.libraries.toml`), each entry includes:

| Field                  | Type   | Description                                                                              |
|------------------------|--------|------------------------------------------------------------------------------------------|
| `name`                 | String | Contract/library name from config                                                        |
| `hash`                 | String | Library code hash (hex)                                                                  |
| `code`                 | String | Library code BoC (base64)                                                                |
| `account`              | String | Library account address on-chain                                                         |
| `duration`             | Number | Initial funded duration in seconds (used for publish/top-up estimation)                  |
| `network`              | String | Network where the library is used (`testnet` / `mainnet` / `localnet` / `custom:<name>`) |
| `timestamp`            | String | Publication time (RFC3339)                                                               |
| `last_topup_timestamp` | String | Last successful top-up time (RFC3339), used for `Remaining` calculation                  |
| `bits`                 | Number | Library code size in bits                                                                |
| `cells`                | Number | Library code size in cells                                                               |

`last_topup_timestamp` is required and is used by `acton library info` to estimate remaining storage runway.
