Coverage tells which lines ran â€” not whether tests would catch a bug in them. This page introduces two complementary techniques:

- **Mutation testing**: deliberately breaks the contract code and checks whether tests notice
- **Fuzz testing**: generates random inputs to probe edge cases

## Mutation testing

The mutation engine makes small, deliberate changes â€” for example, changing `option0 += 1` to `option0 -= 1` â€” then runs the test suite. If all tests still pass after a mutation, that mutation is a **survivor**: tests did not catch the change. Survivors point to gaps that line coverage misses.

One mutation survived: changing `Errors.AlreadyVoted` to `Errors.NotFromPoll` in Voter's assert. Tests checked that a duplicate vote is rejected â€” but none checked the specific exit code. Add one:

```tolk title="tests/poll.test.tolk"
get fun `test voter exits with AlreadyVoted on duplicate`() {
    val (poll, _, alice, _) = setupTest();

    // First vote deploys and sets hasVoted = true.
    val res1 = poll.sendVote(alice.address, 0, { value: ton("0.1") });
    expect(res1).toHaveAllSuccessfulTxs();

    val voterAddr = poll.calcVoterAddress(alice.address);
    val voter     = Voter.fromAddress(voterAddr);

    val directMsg = createMessage({
        bounce: false,
        value:  ton("0.05"),
        dest:   voter.address,
        body:   Vote { option: 0 },
    });
    val res2 = net.send(alice.address, directMsg);
    expect(res2).toHaveFailedTx({
        from:     alice.address,
        to:       voter.address,
        exitCode: Errors.AlreadyVoted as int,
    });
}
```

## Run mutation testing on Poll

```bash
acton test --mutate --mutate-contract Poll
```

Address any survivors. Common ones:

- Mutations to `if (msg.option == 0)` in both `onInternalMessage` and `onBouncedMessage`
- Off-by-one in increment or decrement operations

  Use `--mutation-diff` to test only mutations in lines changed since the main branch â€” useful in CI to skip re-testing stable code.

## Add a fuzz test for the option boundary

```tolk title="tests/poll.test.tolk"
import "@acton/testing/expect"

@fuzzing(min: 0, max: 255)
get fun `fuzz vote option boundary`(option: uint8) {
    val (poll, _, alice, _) = setupTest();

    val msg = createMessage({
        bounce: false,
        value:  ton("0.1"),
        dest:   poll.address,
        body:   Vote { option },
    });
    val res = net.send(alice.address, msg);

    if (option == 0 || option == 1) {
        expect(res).toHaveSuccessfulTx({ from: alice.address, to: poll.address });
    } else {
        expect(res).toHaveFailedTx({
            from:     alice.address,
            to:       poll.address,
            exitCode: Errors.InvalidOption as int,
        });
    }
}
```

```bash
acton test --fuzz-seed 42 -f "fuzz vote option"
```
