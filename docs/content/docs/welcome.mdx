Acton is an all-in-one TON smart contract development toolkit written in Rust.
It combines project scaffolding, build, testing, scripting, wallet and network
operations, verification, linting, formatting, and low-level VM tooling in one
CLI.

<Callout>
    New to Acton? Start with [Installation](/docs/installation), then follow
    the [Quickstart](/docs/quickstart). If you already have a repository, start
    with `acton init` and the [Project initialization](/docs/project-init)
    guide instead of creating a new scaffold.
</Callout>

## What you can do with Acton

- Create a new Tolk project from a ready-made template.
- Build contracts, run tests, and script interactions from one CLI.
- Manage wallets, deploy to the network, and verify published contracts.
- Work locally with a TON-compatible development node, faucet, snapshots, and
  forked state.

## Recommended reading order

<Steps>
    <Step title="Install Acton">
        Start with [Installation](/docs/installation) and make sure
        `acton --version` works in a fresh shell.
    </Step>
    <Step title="Create or open a project">
        Use [Project initialization](/docs/project-init) to decide whether you
        should run `acton new` or `acton init`. Existing repositories do not
        need to be recreated from a template.
    </Step>
    <Step title="Build and test locally">
        Follow the [Quickstart](/docs/quickstart) and the
        [Test Runner](/docs/test-runner) docs to get a reliable local feedback
        loop first.
    </Step>
    <Step title="Prepare deployment">
        Set up wallets, then use the deployment and scripting guides when you
        are ready to send messages or publish contracts.
    </Step>
    <Step title="Move to repeatable workflows">
        Once the basics are working, read the build system, linting, and CI
        sections to make the project reproducible for a team or pipeline.
    </Step>
</Steps>

## Need something specific?

- Want the shortest path to a running project? Go to
  [Quickstart](/docs/quickstart).
- Looking for command syntax? Open [Commands](/docs/commands).
- Need help reading CLI help? Use `acton --help` for the overview,
  `acton help <command>` for the long manual, and `<nested> --help` for exact
  nested subcommand help such as `acton wallet list --help`.
- Working on tests? Start with [Test Runner](/docs/test-runner).
- Need environment and dependency details? Read
  [Acton.toml](/docs/acton-toml) and [Build system](/docs/build-system).
- If setup or path resolution looks wrong, run `acton doctor`.
