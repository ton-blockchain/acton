Acton dev skills are reusable instructions for coding agents working on TON smart contracts. They help the agent choose the right docs, commands, project layout, and safety checks before editing code.

<Callout type="info">
    Skills guide the assistant, but they do not replace local validation. For project changes, still run the relevant Acton commands such as `acton build`, `acton test`, `acton check`, and `acton fmt --check`.
</Callout>

## Available skills

- `acton`: Use for Acton CLI workflows: installation and updates, project scaffolding, `Acton.toml` configuration, builds, single-file compilation, wrapper generation, tests, scripts, wallets, verification, RPC inspection, libraries, linting, formatting, hooks, IDE support, and troubleshooting.
- `tolk`: Use for writing, reviewing, debugging, and testing Tolk smart contracts. It focuses on typed storage and message schemas, union dispatch, auto-serialization, typed maps, explicit bounce and fee behavior, standard getter shapes, and focused contract tests.
- `func2tolk`: Use when porting FunC contracts to idiomatic Tolk. It combines `acton func2tolk` with a compatibility pass for TL-B layouts, opcodes, errors, send modes, bounce behavior, typed Tolk refactors, wrappers, tests, scripts, and verification.
- `ton-blockchain`: Use for TON ecosystem context and standards. It sends the agent to primary TON Docs for TL-B, TVM, cells, BoC, Jettons, wallets, TON Connect, workchains, shardchains, liteservers, and other blockchain concepts.

Install the Acton skill globally using `npx skills`:

```bash
npx skills add https://github.com/ton-blockchain/acton-contracts/tree/skills/skills/
```

Install the other skills individually the same way by changing the final path:

```bash
npx skills add https://github.com/ton-blockchain/acton-contracts/tree/skills/skills/tolk
```

Or for the specific agent:

```bash
npx skills add https://github.com/ton-blockchain/acton-contracts/tree/skills/skills/ton-blockchain -g -a codex -y
npx skills add https://github.com/ton-blockchain/acton-contracts/tree/skills/skills/func2tolk -g -a claude-code -y
```

After installation, mention a skill by name in your coding-agent prompt, for example `$acton`, `$tolk`, `$ton-blockchain`, or `$func2tolk`.
