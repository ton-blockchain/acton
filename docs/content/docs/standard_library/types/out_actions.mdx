Module defining out actions.

This module provides structures and helpers for working with "out actions" generated by smart contracts.
Out actions include sending messages, updating code, reserving currency, and changing libraries.

Examples:
```tolk
// Retrieve actions after contract execution
val actions = vm.outActions();

// Check the first action
val firstAction = actions.at(0);
if (firstAction is OutActionSendMessage) {
    println("Sending message with mode: {}", firstAction.mode);
}

// Get specific message body
val transfer = actions.getSendMessageBodyAt<Transfer>(0);
if (transfer != null) {
    println("Transferred: {:ton}", transfer.amount);
}
```

## Definitions

## `OutList`

```tolk
struct OutList {
    /// Link to previous out-list node.
    prev: Cell<OutList>
    /// Serialized out action stored in this node.
    action: OutAction
}
```

Represents a node in the out list.
This is an internal linked-list node, you should not use it directly.

`prev` is stored in the list node itself and points to the previous list node.
It is not a field of any `OutAction*` payload.

## `OutActionSendMessage.loadMessage`

```tolk
fun OutActionSendMessage.loadMessage<Msg>(self): MessageRelaxed<Msg>
```

Loads the message from the out message cell.

Note that you should provide the message type as a generic parameter.

If you want to access the body of the message, use [OutActionSendMessage.loadBody](#outactionsendmessageloadbody) method.

Example:
```tolk
val out_actions = vm.outActions();
val action = out_actions.at(0);
if (action is OutActionSendMessage) {
    val message = action.loadMessage<IncreaseCounter>();
    val body = message.loadBody();
    println(body.increaseBy);
}
```

<SourceCodeLink href="https://github.com/ton-blockchain/acton/blob/master/lib/types/out_actions.tolk#L67" />

## `OutActionSendMessage.loadGenericMessage`

```tolk
fun OutActionSendMessage.loadGenericMessage(self): MessageRelaxedGeneric
```

Loads the generic message from the out message cell.
Unlike [OutActionSendMessage.loadMessage](#outactionsendmessageloadmessage), this method does not require type of the message to load.
Instead it loads the message with body as a slice. Use this method if don't know the type of the message.

Example:
```tolk
val out_actions = vm.outActions();
val action = out_actions.at(0);
if (action is OutActionSendMessage) {
    val message = action.loadGenericMessage();
    val opcode = message.loadOpcode();
    println(opcode);
}
```

## `OutActionList.at`

```tolk
fun OutActionList.at(self, i: int): OutAction
```

Returns the out action at the given index.

NOTE: this list is reversed, so the last action is at index 0.

Fails with [Assert.fail](../../testing/assert/#assertfail) if the stored tuple element cannot be decoded as [OutAction](#outaction).

Example:
```tolk
val out_actions = vm.outActions();
val action = out_actions.at(0);
println(action.kind());
```

<SourceCodeLink href="https://github.com/ton-blockchain/acton/blob/master/lib/types/out_actions.tolk#L192" />

## `OutActionList.getSendMessageBodyAt`

```tolk
fun OutActionList.getSendMessageBodyAt<Msg>(self, i: int): Msg?
```

Returns the body of the send message at the given index.

Note that you should provide the message type as a generic parameter.

If the action is not a send message, returns null.
If the action is a send message but `Msg` does not match actual body type, this call throws
(for example with exit code 63 from typed message decoding).

Example:
```tolk
val out_actions = vm.outActions();
val body = out_actions.getSendMessageBodyAt<IncreaseCounter>(0);
if (body != null) {
    println(body.increaseBy);
}
```

## `vm.parseOutActions`

```tolk
fun vm.parseOutActions(cell: cell): OutActionList
```

Parses the out actions from the raw cell (like C5 register).

If [cell](#vmparseoutactions) is empty (no bits and no refs), returns an empty tuple.
Fails with [Assert.fail](../../testing/assert/#assertfail) if list node layout is malformed.

Example:
```tolk
val out_actions = vm.parseOutActions(vm.getC5());
println(out_actions.size());
```
